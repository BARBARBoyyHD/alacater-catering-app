# Alacater Color Guidelines & Rules

## Design System Overview

This document defines the color palette, usage rules, and design tokens for the Alacater React Native application. All UI components should adhere to these guidelines for consistency.

---

## Primary Color Palette

### Brand Colors

| Color Name | Hex Code | Usage |
|------------|----------|-------|
| **Primary Orange** | `#FF7B00` | Main brand color, CTAs, active states, highlights |
| **Primary Orange Light** | `#FFF5EB` | Background for selected states, subtle accents |
| **Primary Orange Dark** | `#E56E00` | Hover states, pressed states (if needed) |

### Secondary Colors

| Color Name | Hex Code | Usage |
|------------|----------|-------|
| **Success Green** | `#1B5E3A` | Savings badges, delivered status, positive indicators |

| Color Name | Hex Code | Usage |

## Color Usage Rules

### 1. Primary Actions (CTAs)

```typescript
// âś… Correct
backgroundColor: '#FF7B00'  // Primary buttons
color: '#FFFFFF'            // Text on primary buttons

// âťŚ Wrong
backgroundColor: '#FF3B30'  // Don't use red for primary actions
backgroundColor: '#1B5E3A'  // Don't use green for primary actions
```

**Examples:**
- "Pesan Sekarang" button
- "Lanjut Pembayaran" button
- "Hubungi Merchant" button

### 2. Status Indicators

```typescript
// Status badges
shipping:   '#4A6FA5'  // Blue - In progress
delivered:  '#1B5E3A'  // Green - Completed
pending:    '#FFB800'  // Yellow - Waiting
cancelled:  '#FF3B30'  // Red - Error/Cancelled
```

### 3. Pricing Display

```typescript
// Current/Discounted price
color: '#FF7B00'        // Orange for final price
fontWeight: 'bold'

// Original price (crossed out)
color: '#999999'        // Gray for original price
textDecorationLine: 'line-through'

// Free/Discount labels
color: '#FF7B00'        // Orange for "GRATIS" or discount amounts
fontWeight: '600'
```

### 4. Interactive States

```typescript
// Default state
backgroundColor: '#FFFFFF'
borderColor: '#E5E5E5'

// Active/Selected state
backgroundColor: '#FFF5EB'  // Light orange background
borderColor: '#FF7B00'      // Orange border

// Disabled state
borderColor: '#CCCCCC'
opacity: 0.5
```

### 5. Text Hierarchy

```typescript
// Headings (H1, H2)
color: '#333333'
fontWeight: 'bold' | '600'

// Body text
color: '#666666'

// Meta information (timestamps, labels)
color: '#999999'

// Links/Action text
color: '#FF7B00'
fontWeight: '600'
```

### 6. Notification Badges

```typescript
// Notification count badges
backgroundColor: '#FF3B30'  // Red
color: '#FFFFFF'
```

### 7. Savings/Discount Badges

```typescript
// Savings banner background
backgroundColor: '#1B5E3A'  // Green
color: '#FFFFFF'
```

## Accessibility Guidelines

### Contrast Ratios

Ensure all text meets WCAG 2.1 AA standards:

| Text Size | Minimum Contrast Ratio |
|-----------|----------------------|
| Normal (< 18pt) | 4.5:1 |
| Large (â‰Ą 18pt) | 3:1 |

### Approved Combinations

âś… **Passes AA:**
- `#333333` on `#FFFFFF` (15.3:1)
- `#FFFFFF` on `#FF7B00` (4.7:1)
- `#FFFFFF` on `#1B5E3A` (7.2:1)
- `#666666` on `#FFFFFF` (5.7:1)

âťŚ **Fails AA:**
- `#999999` on `#F5F5F5` (1.4:1) - Don't use for important text
- `#FF7B00` on `#FFFFFF` (2.1:1) - Only use for large text or with border

### Color Blindness Considerations

- Never use color alone to convey information
- Always pair status colors with icons or text labels
- Example: Use "âś“" icon with green, "âś•" icon with red

## Quick Reference Do's and Don'ts

### âś… DO

- Use `#FF7B00` for all primary actions and CTAs
- Use `#1B5E3A` for savings and success states
- Use `#FFF5EB` for selected/active card backgrounds
- Use `#333333` for headings and important text
- Use `#666666` for body text
- Use `#999999` for timestamps and meta information
- Keep sufficient contrast between text and background

### âťŚ DON'T

- Don't use red (`#FF3B30`) for primary actions
- Don't use multiple colors for the same purpose
- Don't use color alone to indicate status
- Don't use light gray text on light backgrounds
- Don't deviate from the defined palette without approval
- Don't use pure black (`#000000`) - use `#333333` instead
