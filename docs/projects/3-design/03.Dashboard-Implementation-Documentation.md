# Design Implementation Documentation - Order Management Dashboard

## 1. Overview
Implemented the **Web-based Order Management Dashboard** for catering providers as part of the `web-order-management-dashboard` OpenSpec change. The implementation covers the provider-side interface for managing B2B orders, status transitions, and viewing delivery schedules.

## 2. Technical Stack
- **Framework**: Next.js 16 (App Router)
- **Data Fetching/Caching**: TanStack Query (v5)
- **Styling**: Tailwind CSS
- **Icons**: `react-icons` (Feather Icons set)
- **Database**: PostgreSQL (via Supabase)
- **ORM**: Prisma 6.3.0
- **Real-time**: Supabase Realtime

## 3. Architecture Details

### 3.1 Data Flow
- **Fetching**: All order lists and details are fetched using custom hooks (`useOrders`, `useOrderDetails`) in `next-app/src/hooks/use-orders.ts`. 
- **Caching**: TanStack Query handles caching with a `staleTime` of 5 minutes.
- **Real-time Sync**: The `useOrderRealtime` hook subscribes to the `Order` table in Supabase. On `INSERT` or `UPDATE`, the hook automatically invalidates the `orders` query cache, triggering a re-fetch of the order list.

### 3.2 Backend Implementation
- **API Routes**: Located at `next-app/src/app/api/orders/`. 
  - Routes are secured using Supabase authentication.
  - The `PATCH` endpoint for status updates is managed by `OrderService.updateStatus`, which performs state transition validation and logs status history in the `OrderStatusHistory` table.
- **Database Schema**: 
  - Added `OrderStatusHistory` model for auditing.
  - Updated `OrderStatus` enum to support status lifecycle (PENDING -> DELIVERED).

### 3.3 UI Implementation
- **Dashboard Layout**: Uses a responsive `DashboardLayout` with a toggleable sidebar and `TopNav`.
- **Components**:
  - `OrderTable`: Handles data display, status badge coloring, and action buttons.
  - `OrderDetailModal`: Provides a full view of order details, subscription progress (visualized with progress bars), and status history.
  - `OrderFilters`: Enables filtering orders by status and searching by order number or customer name.
- **Responsive Design**: The UI is optimized for both desktop and mobile, with interactive navigation, mobile-friendly table scrolling, and responsive modals.

## 4. Key Design Decisions

| Decision | Rationale |
| :--- | :--- |
| **TanStack Query** | Existing project standard for server state; ensures consistent caching across the app. |
| **Supabase Realtime** | Ensures the dashboard is "live" without manual page refreshes or polling. |
| **Atomic Updates** | `OrderService` uses Prisma transactions to ensure the order status update and history logging always succeed together. |
| **Icon Set** | Migrated to `react-icons/fi` to maintain a consistent, modern icon library. |

## 5. Security
- **Auth Verification**: All API routes verify the requester's identity via `supabase.auth.getUser()`.
- **Row-Level Security**: All database interactions are governed by Supabase RLS policies (in addition to Prisma application-layer checks).
