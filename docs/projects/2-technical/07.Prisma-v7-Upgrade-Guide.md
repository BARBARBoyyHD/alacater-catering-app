# Documentation: Prisma ORM v7 Constructor Error

## 1. Issue Description
When upgrading to Prisma ORM v7, the application encountered a `Runtime PrismaClientConstructorValidationError`.
**Error Message**: `Using engine type "client" requires either "adapter" or "accelerateUrl" to be provided to PrismaClient constructor.`

This error occurs because Prisma ORM v7+ introduced a breaking change where the engine now requires an explicit driver adapter (e.g., for PostgreSQL, MySQL, etc.) or Prisma Accelerate for connection management, rather than handling raw TCP connections internally.

## 2. Resolution
The issue was resolved by implementing the `pg` driver adapter, which is the recommended approach for direct PostgreSQL connections.

### 2.1 Dependencies Installed
- `@prisma/adapter-pg`
- `pg`

### 2.2 Updated Implementation (`next-app/src/lib/prisma.ts`)
The `PrismaClient` initialization was updated to inject the `PrismaPg` adapter:

```typescript
import { PrismaClient } from '../generated/prisma'
import { PrismaPg } from '@prisma/adapter-pg'
import { Pool } from 'pg'

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined
}

// Ensure pool is created with the connection string
const connectionString = process.env.DATABASE_URL
const pool = new Pool({ connectionString })
const adapter = new PrismaPg(pool)

export const prisma =
  globalForPrisma.prisma ??
  new PrismaClient({
    adapter,
    log: process.env.NODE_ENV === 'development' ? ['query', 'error', 'warn'] : ['error'],
  })

if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma
```

## 3. Prevention & Guidelines
- **Always update Prisma initialization**: When upgrading major Prisma versions, verify `PrismaClient` constructor requirements in the [official upgrade guide](https://www.prisma.io/docs/guides/upgrade-prisma-orm/v7).
- **Driver Adapters**: For direct DB connections, always use the appropriate driver adapter (`@prisma/adapter-pg` for Postgres, `@prisma/adapter-neon` for Neon, etc.).
- **Environment Variables**: Ensure `DATABASE_URL` is correctly configured in the `.env` file to be used by the `pg.Pool`.
